1 Pyxel Editor

1.1 The Editor

Pyxel has a great built in tool to create resources or assets for our creations, Pyxel Editor. We can create sprites, backgrounds, sounds and music using Pyxel Editor.

Assets are the things our game uses that isn’t code. Any graphics, sounds, pictures can all be considered as assets.

Make sure you’re in the correct pyxel directory and have a virtual environment enabled.

As this is a new project, using Powershell or the terminal, start in our pyxel directory and create a new directory called game. To move back up a level in the directories, you can type cd ...

mkdir game
cd game

Now open Pyxel Editor with the pyxel edit command. game will be the name of our file.

pyxel edit game

The Pyxel Editor opens in the image tab, shown as the person in white in the top left.

The menu bar in blue at the top has tabs for image, tilemap, sound and music. There are also undo, redo and save buttons and a context output.

1.2 Image

In the image tab, the context output will show the current coordinates, watch them change as you move the pointer over the boxes below.

Centre left has a zoomed in view of our image bank, it has a grid of squares. Each square is eight pixels wide and eight high. Below the zoomed in view is the colour picker and tool bar. To the right is a zoomed out view and below it is the image bank picker.

There are three image banks available, 0, 1 and 2. We will add many images to each bank, pyxel needs to know the image bank and coordinates of an image to use it. Perhaps think of each image bank as a piece of paper, we will draw many images on each piece of paper and tell pyxel which piece of paper and where in the piece of paper to look for the image to use.

1.3 Sprite

Lets draw a sprite in image bank 0 at coordinates (0,0).

Make sure the image bank is 0 and the scroll bars are all the way up and left. Choose yellow, then the filled circle. Move the pointer to the top left of the zoomed in box, it should say (0,0) in the context output. Click and drag to create a yellow circle that fits in the top left square in the grid.

Click on the downwards arrow in the menu bar to save, then close the Pyxel Editor window.

We can see the newly created file game.pyxres with the ls command to list the directory contents.

ls

In Powershell, the result should look like this. It will be similar but simpler in a Linux terminal.

Mode                 LastWriteTime         Length Name
----                 -------------         ------ ----
-a----        01/10/2025     17:54            330 game.pyxres

If you don’t see the game.pyxres file in the output, you may not have saved in the Pyxel Editor. Open the Pyxel Editor again with pyxel edit game and try again.

1.4 Code

The code to display our sprite uses two elements not in our hello_world.py example. We need to load the game.pyxres file and use .blt() to draw the sprite image.

Start VSCode with code . if it’s not already running. If it is already running, use the file browser to the left to open the game directory.

Create a new file by hovering over the file explorer and clicking the new file icon + or use the keyboard shortcut ctrl+n. Name the file game.py and add the following code.

game.py

 1import pyxel
 2
 3## this is init which runs once at the start to set screen size, title, frame rate, etc
 4pyxel.init(
 5    width=120,
 6    height=160,
 7    title="Game",
 8    fps=30,
 9    quit_key=pyxel.KEY_X,
10)
11
12## load the pyxres file
13pyxel.load("game.pyxres") 
14
15## update handles inputs
16def update():
17    pass
18
19## draw draws the frames
20def draw():
21    ## clear the screen between each frame
22    pyxel.cls(pyxel.COLOR_BLACK)
23    ## blt() to draw an image
24    pyxel.blt(
25        x=8,   # x position to draw image in frame
26        y=8,   # y position to draw image in frame
27        img=0,  # image bank number for the source image
28        u=0,    # first coordinate of the source image
29        v=0,    # second coordinate of the source image
30        w=8,    # width of the source image
31        h=8     # height of the source image
32    )
33
34pyxel.run(update, draw)

There are a few more comments in this code, comments help us remember what everything does!

The x position is the number of pixels from the left of the window. The y position is the number of pixels from the top of the window.

Save the file game.py using the File menu or keyboard shortcut ctrl+s.

1.5 Run

Run the app, make sure everything is saved, then either click the play icon top right in VSCode or enter the pyxel run command from the terminal:

pyxel run game

This is our sprite image displayed in the window.
Hit X or close the window to quit.

If you only get a black screen or only part of the circle is showing, you may not have drawn the image in the correct location, in image bank 0 at (0,0) in the Pyxel Editor. Go back to the Sprite section, and try again.

If the sprite is in image bank 0 and at (0,0), but it’s still not there, check that “game.pyxres” is correct in the load() command and that the game.pyxres file is named correctly in the game directory.

1.6 What’s Happening?

Our code still has init(), update() and draw(), but there’s a little more to it.

We’ve configured a few more optional parameters in init. They are title and quit key, which we have made X. Oh, and we made the window a different size.

 4pyxel.init(
 5    width=120,
 6    height=160,
 7    title="Game",
 8    fps=30,
 9    quit_key=pyxel.KEY_X,
10)

Next, we import the game.pyxres file. This is important, the images won’t show without it.

13pyxel.load("game.pyxres")

Then in draw() we have the pyxel.blt() method which will fetch an image from our game.pyxres file and draw it to the frame.

24    pyxel.blt(
25        x=8,  # x position to draw image in frame
26        y=8,  # y position to draw image in frame
27        img=0,  # image bank number for the source image
28        u=0,  # first coordinate of the source image
29        v=0,  # second coordinate of the source image
30        w=8,  # width of the source image
31        h=8,  # height of the source image
32    )
  • x and y are the position of the image in the frame, this will be the top left corner of the image
  • img is the image bank in the .pyxres file where we want our source image to come from
  • u and v are the top left coordinates of the source image, in the image bank
  • w and h are the number of pixels accross and down the image occupies

We drew our sprite image in image bank 0, at coordinates (0,0) and it was 8 pixels wide and 8 pixels high, the size of the box in the grid.

Let’s create a bug in our code! Change w in blt() to 4 and running the code again. You should only see half the circle drawn, as only 4 pixels from the left of the source image are being fetched. Change the code back to 8 when you have finished.

1.7 Key Points

  • The pyxel editor saves to a .pyxres file
  • The image tab in the Pyxel Editor has 3 image banks, 0, 1 and 2
  • Each bank shows a grid of squares 8 x 8 pixels
  • Each pixel in the image bank has coordinates u and v (u,v)
  • pyxel.blt() is the method used to fetch an image and draw it in a frame.
