1 Installation

1.1 Windows

1.1.1 Python

First we need to install Python, if you don’t have that installed already.

Go to the Microsoft Store and search for Python. Press the install button to install and follow any on screen instructions.

Open Powershell, try pressing the windows key then typing powershell it should come up.

Top Tip - Install Windows Terminal from the Microsoft Store for a better terminal experience.

Verify you have Python installed by typing:

python --version

and press enter. You should see Python followed by the version number, for example:

Python 3.12.0

If not, try the install steps again.

Next check PIP (the Python Package Installer) was installed with Python. Enter the command:

pip --version

You should see pip followed by the version number and the install location, something like this:

pip 24.3.1 from C:\Users......

If not, enter the following to bootstrap the pip that was installed with Python:

python -m ensurepip --default-pip

and check it worked by trying pip –version again:

pip --version

If either Python or pip didn’t install you cannot go further. Seek help to complete the installs.

1.1.2 Set Windows Execution Policy

You can skip this step if you already have the Execution Policy set.

Close Powershell. We are going to open Powershell again, but this time as an administrator.

If you don’t have administrator access on the system you are using, contact the person who has administrator access for this step.

Press the Windows key and type powershell, the option to open as administrator will show if you have access. Click this and click ‘Yes’ to the warning that pops up.

To avoid having to do this everytime you want to code, I suggest entering the following command to set the execution policy to remote signed for the current user.

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

Close Powershell again and continue below to install VSCode.

Full details of setting execution policies can be found here.

1.1.3 Install VSCode

We will use VSCode in this tutorial to simplify cross Operating System instructions.

Either go to the Microsoft Store, search for vscode and install,

or go to

https://code.visualstudio.com/download

and download the VSCode installer for your system.

Find the download and run the installer, follow the on screen instructions.

1.1.4 Install Pyxel

Open Powershell again as normal (not as an administrator), we will make a new directory for our Pyxel projects. I will use a Projects and a pyxel directory here, you can choose what you want but a pyxel directory is useful.

Where you place the directory is up to you. A good place in Windows would be
C:\Users\**your_username**\

mkdir Projects
cd Projects
mkdir pyxel
cd pyxel

Now we will create a Python virtual environment to run all our pyxel code in, this is good practice as it will keep everything separate from the usual system stuff. Our virtual environment will be called venv.

A Python virtual environment separates PIP packages from system packages, this is a safer way to run Python code as the PIP packages cannot change any system packages that may already be installed.

python -m venv venv

And then activate the virtual environment:

venv/Scripts/activate

You should now have (venv) PS C:\……..> showing in the command line.

Now enter:

echo 'pyxel' > requirements.txt

this creates a text file called requirements.txt with the word pyxel in it.

Next we use pip install to install the PIP pyxel package from requirements.txt. Enter:

pip install -r requirements.txt

You will hopefully see the install notices on screen and pyxel being installed.

Note, pyxel will only work inside the virtual environment venv we have created, we will have to start the virtual environment each time we come back.

Validate pyxel has installed by entering:

pyxel

You should see a helpful list of commands like this:

Pyxel 2.3.1, a retro game engine for Python
usage:
    pyxel run PYTHON_SCRIPT_FILE(.py)
    pyxel watch WATCH_DIR PYTHON_SCRIPT_FILE(.py)
    pyxel play PYXEL_APP_FILE(.pyxapp)
    pyxel edit [PYXEL_RESOURCE_FILE(.pyxres)]
    pyxel package APP_DIR STARTUP_SCRIPT_FILE(.py)
    pyxel app2exe PYXEL_APP_FILE(.pyxapp)
    pyxel app2html PYXEL_APP_FILE(.pyxapp)
    pyxel copy_examples

Now we can open VSCode in the current directory with:

code .

Once VSCode has opened, find ‘Extensions’ on the left hand side. Search for Python and install the top Python extension, this will give us code highlighting and other useful features.

1.1.5 Well Done!

Python, Pyxel and VSCode are now installed and you are ready to move on to the Hello World tutorial. Follow the link below.

1.2 Linux Install

1.2.1 Install Python

You will likely already have Python installed.

Open terminal and verify you have Python installed by typing:

python3 --version

and press enter. You should see Python followed by the version number, for example:

Python 3.12.0

If not, use the appropriate package manager for your distibution to install Python. For example this may be:

apt install Python3

Next check PIP (the Python Package Installer) was installed with Python. Enter the command:

pip --version

You should see pip followed by the version number, for example:

pip 24.3.1

If not, enter the following to bootstrap the pip that was installed with Python:

python3 -m ensurepip --default-pip

and check the success with:

pip --version

1.2.2 Install Pyxel

In the terminal, make a new directory. I will use a Projects and a pyxel directory here, you can choose what you want but a pyxel directory is useful.

Where you make the directory is up to you. A good place in Linux would be
/home/**username**/

mkdir Projects
cd Projects
mkdir pyxel
cd pyxel

Now we will create a Python virtual environment to run all our pyxel code in, this is good practice as it will keep everything seperate from the usual system stuff. Our virtual environment will be called venv.

python3 -m venv venv

And the activate the virtual environment:

source venv/bin/activate

You should now have (venv)/home/**username**/…….. showing in the command line.

Now enter:

echo 'pyxel' > requirements.txt

this creates a text file called requirements.txt with the word pyxel in it.

Next we use pip install to install the PIP pyxel package from requirements.txt. Enter:

pip install -r requirements.txt

You will hopefully see the install notices on screen and pyxel being installed.

Note, pyxel will only work inside the virtual environment venv we have created, we will have to start the virtual environment each time we come back.

Validate pyxel has installed by entering:

pyxel

You should see a helpful list of commands like this:

Pyxel 2.3.1, a retro game engine for Python
usage:
    pyxel run PYTHON_SCRIPT_FILE(.py)
    pyxel watch WATCH_DIR PYTHON_SCRIPT_FILE(.py)
    pyxel play PYXEL_APP_FILE(.pyxapp)
    pyxel edit [PYXEL_RESOURCE_FILE(.pyxres)]
    pyxel package APP_DIR STARTUP_SCRIPT_FILE(.py)
    pyxel app2exe PYXEL_APP_FILE(.pyxapp)
    pyxel app2html PYXEL_APP_FILE(.pyxapp)
    pyxel copy_examples

1.2.3 Install Visual Studio Code

We will use Visual Studio Code in this tutorial to simplify cross Operating System instructions.

Go to

https://code.visualstudio.com/download

and download the VSCode installer for your system.

Find the download and run the installer, follow the on screen instructions.

In the terminal, open VSCode in the current directory with:

code .

In VSCode, find ‘Extensions’ on the left hand side. Search for Python and install the top Python extension, this will give us code highlighting and other useful features.

1.2.4 Well Done!

Python, Pyxel and VSCode are now installed and you are ready to move on to the Hello World tutorial. Follow the link below.

Hello World Tutorial

1.3 Uninstall

Don’t like Pyxel? All you need to do to uninstall is delete the pyxel directory where we created the venv virtual environment. To uninstall Python and VSCode use your usual system methods to uninstall programs and applications.
