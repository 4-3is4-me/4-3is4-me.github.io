1 Movement and Controls

1.1 Inputs

For our game to be playable we must have a way to accept and handle inputs from the player. Thankfully, Pyxel has a way we can do this easily. We are going to extend our game code to move the sprite around the window when the keyboard arrow keys are pressed.

1.2 Code

Make sure you are in the correct pyxel/game directory and have a virtual environment enabled.

To handle the input we will need to use a couple of variables that will change when a key is pressed. These variables are referenced in pyxel.blt() to draw the sprite image in a new location.

game.py

 1import pyxel
 2
 3player_x = 8
 4player_y = 8
 5
 6## this is init which runs once at the start to set screen size, title, frame rate, etc
 7pyxel.init(
 8    width=120,
 9    height=160,
10    title="Game",
11    fps=30,
12    quit_key=pyxel.KEY_X,
13)
14
15## load the pyxres file
16pyxel.load("game.pyxres")
17
18## update handles inputs
19def update():
20    global player_x
21    global player_y
22    ## player inputs
23    if pyxel.btn(pyxel.KEY_RIGHT):
24        player_x += 8
25    if pyxel.btn(pyxel.KEY_LEFT):
26        player_x -= 8
27    if pyxel.btn(pyxel.KEY_UP):
28        player_y -= 8
29    if pyxel.btn(pyxel.KEY_DOWN):
30        player_y += 8
31
32## draw draws the frames
33def draw():
34    ## clear the screen between each frame
35    pyxel.cls(pyxel.COLOR_BLACK)
36    ## blt() to draw an image
37    pyxel.blt(
38        x=player_x,  # x position to draw image in frame
39        y=player_y,  # y position to draw image in frame
40        img=0,  # image bank number for the source image
41        u=0,  # first coordinate of the source image
42        v=0,  # second coordinate of the source image
43        w=8,  # width of the source image
44        h=8,  # height of the source image
45    )
46
47pyxel.run(update, draw)

Save the file game.py using the File menu or keyboard shortcut ctrl+s.

1.3 Run

Run the app, make sure everything is saved, then either click the play icon top right in VSCode or enter the pyxel run command from the terminal:

pyxel run game

Try using the arrow keys to move around. Hopefully the circle will respond to the inputs you make.
Hit X or close the window to quit.

If your sprite is moving in the wrong directions, remember that x and y are both 0 in the top left corner of the frame. So to move up we decrement (reduce) y.

We have a bit of a problem; our sprite can move off the sides of the frame and get lost! Weâ€™ll fix that at a later point.

1.4 Whatâ€™s Happening

Weâ€™ve added some code to update() to handle the key presses and change some variables.

Those variables, player_x and player_y are created at the top, this makes their scope global.

3player_x = 8
4player_y = 8

Variables with a global scope mean they can be referenced anywhere in the rest of the code. Itâ€™s not considered best practice in Python to use global variables, but it is the simplest method here.

update() now has the code for the inputs. First we say that the variables player_x and player_y are the global variables we created earlier. Without using the global keyword, Python would create new variables that would only work inside update().

18def update():
19    global player_x
20    global player_y

Then we have several if statements that check if a button is pressed. The btn() method handles this, it returns True if the specified key is pressed. If it is, then we increment or decrement player_x or player_y by 8.

Pressing the right arrow key, pyxel.KEY_RIGHT makes player_x increase by 8 pixels. When the sprite image is next drawn by draw(), it will appear 8 pixels over to the right. Likewise for the other directions.

22    ## player inputs
23    if pyxel.btn(pyxel.KEY_RIGHT):
24        player_x += 8
25    if pyxel.btn(pyxel.KEY_LEFT):
26        player_x -= 8
27    if pyxel.btn(pyxel.KEY_UP):
28        player_y -= 8
29    if pyxel.btn(pyxel.KEY_DOWN):
30        player_y += 8

Remember that update() and draw() are running every frame, so player_x and player_y are changing fast, in this case up to 30 times per second as we have fps=30 in init().

And finally, we have made a change in blt(), so that x and y location of the sprite image are the player_x and player_y variables. This changes the location to draw the image in the frame when player_x and player_y are changed by inputs handled in update()..

38        x=player_x,  # x position to draw image in frame
39        y=player_y,  # y position to draw image in frame
